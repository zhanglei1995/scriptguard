## 基础信息

**Base URL**: `http://localhost:3000/api/v1`

**认证方式**: Bearer Token (JWT)

```
Authorization: Bearer <your-jwt-token>
```

## 认证接口

### POST /auth/register

注册新用户。

**请求体**:

```json
{
  "email": "user@example.com",
  "password": "secure-password",
  "name": "用户名"
}
```

**响应** `201 Created`:

```json
{
  "id": "uuid",
  "email": "user@example.com",
  "name": "用户名",
  "createdAt": "2024-01-01T00:00:00Z"
}
```

### POST /auth/login

用户登录。

**请求体**:

```json
{
  "email": "user@example.com",
  "password": "secure-password"
}
```

**响应** `200 OK`:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "expiresIn": "7d"
}
```

## 脚本管理

### GET /scripts

获取当前用户的脚本列表。

**查询参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` | number | 页码，默认 1 |
| `limit` | number | 每页数量，默认 20 |
| `status` | string | 过滤状态：healthy / warning / critical |

### POST /scripts

注册新的监控脚本。

**请求体**:

```json
{
  "name": "脚本名称",
  "url": "https://greasyfork.org/scripts/12345/example.user.js",
  "checkInterval": 3600,
  "description": "脚本描述"
}
```

### GET /scripts/:id

获取脚本详情。

### PUT /scripts/:id

更新脚本配置。

### DELETE /scripts/:id

删除脚本。

## 健康检测

### GET /scripts/:id/health

获取脚本健康检测历史记录。

**查询参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| `from` | string | 起始时间 (ISO 8601) |
| `to` | string | 结束时间 (ISO 8601) |
| `limit` | number | 返回记录数 |

### POST /scripts/:id/check

手动触发一次健康检测。

## 版本管理

### GET /scripts/:id/versions

获取脚本版本历史。

### GET /scripts/:id/versions/:versionId

获取特定版本详情。

## 告警管理

### GET /alerts

获取告警列表。

**查询参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| `status` | string | unread / read / dismissed |
| `severity` | string | info / warning / critical |

### PUT /alerts/:id/read

标记告警为已读。

### PUT /alerts/:id/dismiss

忽略告警。

## 通知渠道

### GET /channels

获取通知渠道列表。

### POST /channels

创建通知渠道。

**请求体**:

```json
{
  "type": "webhook",
  "name": "Slack 通知",
  "config": {
    "url": "https://hooks.slack.com/services/..."
  }
}
```

支持的渠道类型：`webhook`、`email`

### DELETE /channels/:id

删除通知渠道。

## 检测规则

### GET /rules

获取检测规则列表。

### POST /rules

创建检测规则。

**请求体**:

```json
{
  "name": "脚本大小变更",
  "type": "size_change",
  "config": {
    "threshold": 0.1
  },
  "enabled": true
}
```

### PUT /rules/:id

更新规则。

### DELETE /rules/:id

删除规则。

## 定时任务

### GET /schedules

获取定时任务列表。

### POST /schedules

创建定时任务。

**请求体**:

```json
{
  "name": "每日检测",
  "cron": "0 9 * * *",
  "enabled": true
}
```

## 数据同步

### POST /sync/push

推送本地数据到云端。

### GET /sync/pull

从云端拉取最新数据。

## 系统端点

### GET /

服务信息。

### GET /health

健康检查。返回 `200 OK` 表示服务正常。

### GET /ready

就绪检查。返回 `200 OK` 表示服务已就绪。

### GET /docs

Swagger API 文档界面。

### GET /metrics

Prometheus 监控指标。
